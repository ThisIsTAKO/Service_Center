# Информационная система "ЛОМБАРД"

## Описание проекта

Приложение для управления базой данных ломбарда с отображением информации в виде таблиц.
Реализовано на Windows Forms с использованием JSON для хранения данных.

## Основные возможности

- **Управление клиентами** - просмотр информации о клиентах ломбарда
- **Управление залоговым имуществом** - каталог предметов, принятых в залог
- **Управление займами** - информация о выданных займах, их статусах и сроках
- **Управление платежами** - история платежей по займам

## Структура базы данных

### 1. Клиенты (Clients)
- ID клиента
- Фамилия, Имя, Отчество
- Паспортные данные
- Телефон
- Адрес
- Дата регистрации

### 2. Залоговое имущество (PledgeItems)
- ID предмета
- Наименование
- Категория (Ювелирные изделия, Электроника, Часы и т.д.)
- Описание
- Оценочная стоимость
- Состояние

### 3. Займы (Loans)
- ID займа
- ID клиента
- ID залогового предмета
- Сумма займа
- Процентная ставка
- Дата выдачи
- Срок погашения
- Статус (Активный, Погашен, Просрочен)
- Уплаченная сумма
- Остаток к оплате

### 4. Платежи (Payments)
- ID платежа
- ID займа
- Дата платежа
- Сумма платежа
- Тип платежа (Основной долг, Проценты)

## Технические характеристики

- **Платформа:** .NET 6.0+ (Windows Forms)
- **Язык программирования:** C#
- **Хранилище данных:** JSON файл (pawnshop_data.json)
- **Архитектура:** Многослойная (Models, Services, UI)

## Файлы

### Models.cs
Содержит классы моделей данных:
- `Client` - модель клиента
- `PledgeItem` - модель залогового предмета
- `Loan` - модель займа
- `Payment` - модель платежа
- `PawnShopDatabase` - контейнер для всех данных

### DataService.cs
Сервис для работы с данными:
- Загрузка и сохранение данных в JSON
- Получение данных по таблицам
- Инициализация тестовых данных

### Form1.cs
Главная форма приложения:
- Создание интерфейса с вкладками
- Отображение данных в таблицах DataGridView
- Цветовое кодирование статусов займов

## Особенности интерфейса

### Цветовое кодирование займов
- **Зеленый** - Активный займ
- **Красный** - Просроченный займ
- **Серый** - Погашенный займ

### Таблицы
- Автоматическое изменение размера столбцов
- Чередующиеся цвета строк для удобства чтения
- Заголовки с темным фоном
- Выделение полной строки при выборе

## Тестовые данные

При первом запуске автоматически создается JSON файл с тестовыми данными:
- 5 клиентов
- 7 предметов залога
- 6 займов
- 6 платежей

## Запуск приложения

1. Откройте проект в Visual Studio 2022 или выше
2. Убедитесь, что целевая платформа установлена как Windows
3. Нажмите F5 для запуска приложения
4. При первом запуске автоматически создается файл с тестовыми данными

## Примеры данных

### Категории залогового имущества
- Ювелирные изделия (кольца, цепочки, серьги)
- Электроника (ноутбуки, смартфоны, игровые консоли)
- Часы (швейцарские механические часы)

### Статусы займов
- **Активный** - займ действует, срок не истек
- **Погашен** - займ полностью погашен
- **Просрочен** - срок займа истек, есть задолженность

## Формат хранения данных

Данные хранятся в файле `pawnshop_data.json` в формате:

```json
{
  "Clients": [...],
  "PledgeItems": [...],
  "Loans": [...],
  "Payments": [...]
}
```
